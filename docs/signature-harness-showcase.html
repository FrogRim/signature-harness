Signature Harness
아키텍처 런타임 증거 검수 경로 한계
AI 코딩 에이전트 완료 검수 하네스

낙관이 아니라 증거로 닫는 Goal Loop.

Signature Harness는 코딩 에이전트의 작업을 Goal, Seed, Active Slice, Trace, Oracle Gate에 묶어 두는 repo-local 하네스입니다. 거짓 완료, 무진행 루프, 약한 증거, 안전하지 않은 resume check, 지나치게 호의적인 에이전트 행동을 잡아내도록 설계했습니다.

95 100점 만점 repo-local 점수
60/60 최신 benchmark trial 통과
12/12 최신 regression trial 통과

주장 상태

  • 구현됨

    런타임 상태, trace, replay, schema, eval fixture, ledger, 권한 검사가 구현되어 있습니다.

  • 검수됨

    UltraQA가 false green 또는 bypass 경로 4개를 실제로 찾고 닫았습니다.

  • MVP

    강한 harness MVP이자 포트폴리오 artifact입니다. 아직 패키징된 vertical product는 아닙니다.

  • 아직 아님

    실제 sandbox adapter, host-level E2E proof, LLM/사람 transcript judge는 아직 주장하지 않습니다.

아키텍처

Control plane은 라우팅과 검증을 맡고, Goal loop는 제한된 slice만 실행합니다. COMPLETE로 가는 유일한 경로는 Oracle입니다.

Goal 사용자 의도를 성공 기준과 제약으로 정규화합니다.
Seed 수락 기준을 포함한 안정적인 실행 계약입니다.
Active Slice 현재 실행 가능한 단위만 배차합니다.
Orchestration pause, gap-fill, recovery, abort를 관제하는 read-only control plane입니다.
실행 Goal loop와 tool은 주장만이 아니라 artifact를 생성합니다.
Trace Ledger, tool call, cost, latency, replay가 기록됩니다.
Oracle 기계적/의미적 증거 gate가 verdict를 결정합니다.
종료 또는 재배차 완료, gap-fill, blocked recovery, evolve, abort 중 하나로 닫습니다.

무엇을 만들었나

이 하네스는 단순 프롬프트 묶음이 아닙니다. 저장소 안에 실행 가능한 런타임 기반과 기계가 읽을 수 있는 계약이 있습니다.

영속 런타임

각 run은 run_manifest.json, state.json, step_ledger.jsonl, handoff.md, replay.json을 남길 수 있습니다.

Trace First

Trace, tool call, 비용, 지연 시간, artifact hash, 승인, state transition을 repo-local evidence로 저장합니다.

Oracle Gate

완료 판정은 artifact-backed evidence가 있어야 가능합니다. 증거가 부족하면 모호한 재시도가 아니라 GAP_FILL이 됩니다.

Tool Contract

schemas/tool_contracts.json은 command input, output, exit code, side effect, 재시도 가능성, failure mode를 선언합니다.

보안 정책

security/policy.json은 read-only, workspace-write, scoped-network, danger-full-access profile을 정의합니다.

Vertical Focus

첫 번째 vertical은 AI coding-agent completion auditing입니다. 용어집, 평가 rubric, failure corpus, demo run을 포함합니다.

최신 검증 증거

현재 점수는 실행 가능한 check와 최신 UltraQA artifact로 방어합니다.

UltraQA에서 닫은 Findings

아래 항목들은 adversarial QA 중 실제로 발견된 false green 또는 bypass 경로이며, 런타임 기반에서 수정되었습니다.

Transition Bypass

record-step이 유효한 transition event 없이 state 변경을 강제할 수 있었습니다. 이제 state-changing record는 canonical transition event가 필요합니다.

얕은 Schema Validation

잘못된 field type을 가진 contract가 통과할 수 있었습니다. 이제 runtime schema validation은 required key와 expected type을 함께 검사합니다.

Missing Artifact False Green

expected artifact path가 없어도 eval task가 통과할 수 있었습니다. 이제 expected artifact는 repo root 기준으로 존재 검증을 합니다.

Scoped Write Gap

scoped_network는 network allowlist만 강제하고 write allowlist는 강제하지 않았습니다. 이제 file write도 profile policy로 검사합니다.

정직한 한계

이 프로젝트가 포트폴리오용 harness MVP로 설득력 있는 이유는 제품 성숙도를 과장하지 않고 한계를 명확히 적기 때문입니다.

98점 이상을 위해 부족한 것 현재 입장
run-resume용 실제 sandbox adapter 아직 구현하지 않았습니다. 안전하지 않은 resume execution은 fail-closed로 막습니다.
Claude Code / Codex host-level E2E orchestration 주장하지 않습니다. 현재 증거는 repo-local 기준입니다.
LLM 또는 사람 기반 transcript judge 아직 구현하지 않았습니다. 현재 eval은 deterministic fixture와 substrate validator 중심입니다.
실제 사용자 run corpus 아직 축적되지 않았습니다. domain corpus는 fixture 기반입니다.

Signature Harness. 강한 harness MVP이자 포트폴리오 artifact입니다. 점수 주장은 95 / 100 repo-local이며, 주장하지 않는 범위도 명시합니다.
