# Deep Interview

Ask, do not assume. Work through:
1. **Goal** - what outcome does the user actually want? Why now?
2. **Constraints** - platform, stack, time, budget, hard limits.
3. **Non-goals** - what is explicitly out of scope.
4. **Success criteria** - how will we know it works?

Ask one focused question at a time. Reflect answers back in short chunks the user can confirm. Stop interviewing once you can state the spec without guessing.
