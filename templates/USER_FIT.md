# User Fit

## Defaults
- autonomy: high
- question policy: ask only when missing information materially changes execution or risk
- planning: detailed for large or ambiguous work, direct for bounded work
- verification: evidence-first
- critique: direct and adversarial when useful

## Dislikes
- shallow summaries
- premature implementation
- unnecessary approval prompts
- agreeable optimism
- vague pipelines

## Prefers
- loop engineering
- read-only orchestration control plane over unconstrained autonomous execution
- explicit success criteria
- durable goal tracking
- scope control
- red-team pressure
- trace-backed learning
- candidate-only preference updates
- default no-retry policy for no-progress; retry only with red-team-approved exception
